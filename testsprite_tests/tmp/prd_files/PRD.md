# وثيقة متطلبات المنتج (PRD) - نظام WaadCare

## 1. نظرة عامة على النظام (System Overview)
**WaadCare (TBA System)** هو نظام متكامل لإدارة نفقات التأمين الطبي، يهدف إلى ربط مقدمي الخدمات الطبية (مستشفيات، عيادات) بشركات التأمين وأرباب العمل والمؤمن عليهم. يقوم النظام بإدارة دورة حياة المطالبات الطبية (Claims) بالكامل، من لحظة تسجيل الزيارة حتى الموافقة النهائية والصرف.

## 2. ميزات النظام الأساسية (Key Features)

### أ. بوابة مقدم الخدمة (Provider Portal)
*   **تسجيل الزيارات (Visit Registration):** تسجيل دخول المؤمن عليه والتحقق من أهليته فورياً.
*   **تقديم المطالبات (Claims Submission):** نظام "قائم على الزيارة" (Visit-Centric) يمنع تقديم أي مطالبة بدون رقم زيارة صحيح.
*   **التسعير التلقائي (Contract-Driven Pricing):** يتم جلب الأسعار آلياً من "عقد مقدم الخدمة" ولا يسمح بالإدخال اليدوي للأسعار لضمان النزاهة.
*   **إدارة المرفقات:** إمكانية رفع التقارير الطبية والفواتير والوصفات.

### ب. إدارة المعجم الطبي (Medical Taxonomy)
*   **التصنيفات الطبية:** إدارة هرمية للأقسام (مختبر، أشعة، جراحة، إلخ).
*   **الخدمات الطبية:** كتالوج خدمات موحد مرتبط بأكواد دولية ومحلية.
*   **تشخيصات ICD-10:** نظام موحد لتصنيف الأمراض والتشخيصات.

### ج. إدارة المؤمن عليهم والسياسات (Members & Benefit Policies)
*   **التحقق من الأهلية (Eligibility):** فحص فوري لسقف التغطية السنوي والمنافع المتاحة.
*   **سياسات المنافع (Benefit Policies):** تحديد نسب التحمل (Co-pay)، حدود الاستهلاك، وشروط التغطية لكل شركة أو فئة.

### د. دورة حياة المطالبة (Claims Workflow)
*   **آلة الحالة (State Machine):** تنتقل المطالبة عبر حالات محددة (مسودة -> مقدمة -> تحت المراجعة -> مقبولة/مرفوضة).
*   **الجمود المالي (Financial Immutability):** بمجرد اعتماد المطالبة، يتم قفل المبالغ المالية لمنع أي تعديل لاحق.
*   **سجل التدقيق (Audit Trail):** تسجيل كل حركة وتعديل مع هوية المستخدم والوقت لضمان الشفافية.

## 3. الهيكل التقني (Technical Architecture)
*   **Backend:** Spring Boot (Java 17), JPA/Hibernate.
*   **Architecture Pattern:** DDD (Domain Driven Design) مع تطبيق صارم للقواعد المعمارية (Architectural Guards).
*   **Frontend:** React.js مع Material UI (MUI).
*   **Security:** RBAC (Role-Based Access Control) باستخدام JWT.

## 4. القوانين المعمارية (Architectural Laws)
1.  **لا مطالبة بدون زيارة:** (visitId is MANDATORY).
2.  **لا خدمات نصية حرة:** يجب اختيار الخدمة من القائمة الموحدة.
3.  **لا تسعير يدوي:** السعر يأتي من عقد مقدم الخدمة المسجل مسبقاً.
4.  **التصنيف إلزامي:** لا يمكن تقديم خدمة طبية غير مرتبطة بتصنيف فني صحيح.
